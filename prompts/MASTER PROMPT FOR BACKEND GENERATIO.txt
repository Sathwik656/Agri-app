MASTER PROMPT FOR BACKEND GENERATION (Node + Express + MongoDB)

Context
You are a senior backend engineer designing a production-ready backend for an AI-Enabled Farmer–Buyer Smart Marketplace Platform.

The system is a two-sided digital marketplace connecting farmers, buyers, investors, and admins, providing:
Real-time market intelligence
AI-assisted crop, yield, and price predictions
Smart farmer–buyer matching
Transparent deal execution.
The backend must be fully functional, frontend-ready, and modular, with clear separation of concerns.

TECHNOLOGY STACK (MANDATORY)
Runtime: Node.js
Framework: Express.js
Database: MongoDB (Mongoose ODM)
Auth: JWT (access + refresh tokens)
Architecture: Layered / Service-Oriented
API Style: REST (optionally hybrid with async jobs)

CORE ARCHITECTURE RULES
1.Follow clean layered architecture:

controllers/
services/
models/
middlewares/
utils/
jobs/ (async / background)
config/
(Improve if necessary)

No business logic inside controllers:
Controllers only:
Validate request
Call service
Return response

AI/ML logic must be abstracted:
Use placeholder / mock prediction logic
All AI calls must go through predictionService / matchingService

Use role-based authorization
Roles: FARMER, BUYER, ADMIN, INVESTOR

Every route must be:
Implemented
Validated
Secured where required
